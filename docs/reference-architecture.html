Node Detail

SQL Generation Priority Chain

① Raw Table Fast Path

No LLM call. Detects raw_* table reference. Returns SELECT *, COUNT(*), or DISTINCT query instantly.

_raw_table_sql()

② Analytical Pattern

Pre-built JOIN queries for common cross-table rankings: customers by cards/spend/disputes, merchants by volume.

_analytical_pattern_sql()

③ Semantic Catalog

Dimension-aware SQL lookup table. 6 metrics × N dimensions = canonical SQL. No hallucination possible.

catalog.py

④ Vanna / Ollama LLM

RAG text-to-SQL (Vanna) or Ollama qwen2.5:7b as final fallback for unknown query patterns.

vanna_service.py

End-to-End Request Flow

① User Input

User types natural language query in chat UI.
POST /chat/query · persona + message
→

② Guardrail

Safety check. Blocks harmful/off-topic queries.
Short-circuits to persist if blocked.
→

③ Planner + Persona

Classifies intent. Loads persona prompt template from personas/*.txt.
↓

④ Semantic Resolver

Scores keywords → matches metric in SEMANTIC_CATALOG.
Extracts dimensions ("by country", "across segments").
Raw table ref → free_form=True
→

⑤ Vanna SQL Node

Priority chain: raw table → pattern → catalog → Vanna → Ollama.
Unknown table names intercepted here → short-circuit with helpful error.
→

⑥ SQL Validator

Parses + normalises SQL with sqlparse.
Sets error state on failure.
↓

⑦ Query Router

Routes to StarRocks (primary) or Trino (federation).
Selects engine from semantic_context.engine.
→

⑧ Query Executor

Executes SQL via mysql-connector on StarRocks port 9030.
Returns rows[] + columns[].
→

⑨ Result Evaluator

Checks row count, quality.
May set _needs_retry=True → loops back to Vanna SQL (max 1 retry).
↓

⑩ Visualization

Recommends chart type (bar, line, pie, heatmap…) based on result shape.
Configures visualization_config for frontend ECharts renderer.
→

⑪ Insight Node

Pure-Python stats: top/bottom, avg, median, spread, trend, concentration.
Skipped for free_form=True queries.
→

⑫ Recommendation

Rule-based actions from statistical findings.
KPI cards, bottleneck flags, highlight actions.
↓

⑬ Response Node

Composes the final natural language answer.
Branches: COUNT result · identity records · ranking list · wide raw scan · metric comparison · time-series · single-row lookup.
→

⑭ Persist Node

Stores report in PostgreSQL.
Returns full JSON state to API → frontend.

Query Intelligence — How Queries Are Routed

Query PatternDetectionRouteExample
Raw table access raw_* name in message Direct StarRocks SELECT "show sample data from raw_merchant"
Count query count|how many + table name SELECT COUNT(*) — no LLM "get count of raw_customer"
Analytical ranking high|most|top intent + cross-table Pre-built JOIN SQL "customers with high number of cards"
Metric query Keyword score ≥ 1 in SEMANTIC_CATALOG Canonical SQL from catalog "fraud rate by country"
Complex/unknown No pattern match Vanna RAG → Ollama LLM "show payment behaviour of high-value customers"
Unknown table name snake_case noun not in known tables Short-circuit: helpful error + fuzzy suggest "show data for ddm_card" → "Did you mean raw_card?"
Blocked query Guardrail LLM check Blocked — no DB access Harmful, off-topic, or injection attempts
